Accomplishments and Preliminary Results

Generating Sub-aperture Images:

We successfully rendered sub-aperture images by modifying project 3 code, which we exported as a .png image. Since each sub-aperture image can be thought of as pinhole camera images taken at a certain lateral offset from the “center”, the program was modified to take in additional arguments that will shift the camera position relative to the camera setting file argument. With this modification, we were able to generate a grid of pinhole camera images a constant offset away from each other which serves as the sub-aperture images for the computational refocusing stage.

Refocusing Sub-aperture Images:

We created a java project that allowed us to refocus the .png images that were created from the modified project 3 code. The paper[1] we read on the light field cameras shows that computational refocusing given sub-aperture images is simply shifting the images and averaging their pixel values. The program calculates the ‘center’ of the grid, and moves the sub-aperture images towards the center proportional to their distance away from the center and the refocus distance.

Building a GUI for Result Comparison:

In our java project, we added GUI functionality that would allow us to toggle the distance between the sensor and the lens. “0” refers to our image focused at infinity, and an increasing number refers to an increased distance between the lens and the sensor plane. This GUI is much more user friendly and renders faster between changes in distance compared to our Project 3 GUI.
