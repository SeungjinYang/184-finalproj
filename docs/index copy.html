Final Project: Light Field Camera

Amanda Nicole, Brendan Sweeney, Seung Jin Yang


Summary

We will increase the functionality of our raytracer from Project 3, we will add light field camera support, where each pixel is now a grid that records radiance from different directions. With this, we will add adjustment and refocusing capabilities. To measure our success, we will provide an interface and compare refocused images with images rendered with the standard thin lens ray tracing.

Problem description

From Project 3, thin lens rendering gives a more photorealistic rendering than a pinhole model as it gives a field of depth. However, once rendered, the image cannot be refocused. Thus, to get the best rendered image, we first made a ballpark measure of conditions by running multiple renderings on low settings before rendering with full settings. By introducing the light field camera, we will be able to render images without selecting a specific focal point on the image using microlens arrays that can make refocus-able images after rendering.

Goal

Our goal is to implement a light field camera by building on the thin lens model in Project 3 using a microlens array as shown in Fig.1. We will do so after reading up on the light field camera model and getting familiar with the mathematical functions associated with it, which we have linked in the “Resources” portion. After doing so, we will want to render images like those in Fig.2, wherein we can refocus the image on the left with a short focal length to render the image on the right with a long focal length.

If time permits, with the model, we will also build an interactive model for comparison with the thin lens and pinhole model, using the Project 3 GUI as a starting point. We want this model to be able to refocus the images by changing the focal length and the lens aperture on the scene. We might also implement more functionality into our GUI that would allow the users to filter the image by toggling brightness, sharpness, contrast, and saturation to produce a more desirable image.

Deliverables

We plan to deliver images generated with plenoptic light field support that be refocused after rendering, images generated with thin lens model for comparison purposes, and an interface to manipulate lens aperture and focus of such image generated by light field support. We also want to generate pictures of some intermediate steps, for example, a grid of sub-aperture images.

We hope to deliver images generated with different lens models and create an (improved) interactive interface to compare results.

Metric of success

A light field camera seeks to manipulate the lens characteristic of the image after rendering. We will measure the effectiveness of light field support by comparing images generated by light field with renderings generated by normal thin lens model at multiple lens configurations.
